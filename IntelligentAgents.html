Unit 1: Introduction to Agent-Based Computing

Overview

This unit introduced the foundational concepts of agent-based computing and explored the trends in technology that have contributed to its rapid growth. We began by understanding what an agent is—an autonomous software program capable of perceiving its environment and acting to achieve its goals. These systems are increasingly used across many industries due to their ability to operate independently and make intelligent decisions.

Unit 3: Agent Architectures

Overview

This unit focused on different types of agent architectures including reactive, deliberative, and hybrid models. Each architecture offers unique ways for agents to process information and respond to their environment, balancing between speed, complexity, and learning capabilities.

  • Reactive Agents: Act based on direct stimuli from the environment with no internal state.
  • Deliberative Agents: Use internal models and planning to select actions.
  • Hybrid Agents: Combine reactive and deliberative layers for flexible decision-making.

Key Learning

  • Identified key characteristics of reactive, deliberative, and hybrid agent architectures.
  • Compared architecture strengths and weaknesses in different use cases.
  • Applied architecture selection to sample problems and environments.

Seminar Summary

We analyzed case studies showing when to use each architecture in real-world applications. Students worked in small groups to match problem scenarios to appropriate agent architectures and present justifications.

Unit 4: Hybrid Agent Architectures

Overview

This unit introduced hybrid architectures that blend reactive and deliberative models. These architectures are designed to balance real-time responsiveness with the ability to reason and plan, making them highly effective in dynamic and complex environments.

  • Described the need for combining architectural models.
  • Examined typical hybrid structures such as Subsumption and InteRRaP.
  • Discussed how agents transition between layers of control.

Key Learning

  • Understood the rationale for hybrid models in agent design.
  • Evaluated real-world use cases that benefit from layered architectures.
  • Constructed basic hybrid architectures using examples and templates.

Unit 5: Agent Communication

Overview

This unit focused on how agents communicate using formal languages and protocols. Key topics included Knowledge Query and Manipulation Language (KQML) and the Foundation for Intelligent Physical Agents (FIPA) standards. We explored how structured communication enables agents to coordinate, negotiate, and share knowledge.

  • Introduced key agent communication languages: KQML and FIPA ACL.
  • Explored performatives and illocutionary acts in message exchange.
  • Discussed ontologies and their role in mutual understanding between agents.

Key Learning

  • Understood the role of standardized communication in multi-agent systems.
  • Analyzed sample agent dialogues using performatives such as inform, request, and propose.
  • Evaluated benefits and limitations of agent communication protocols.

Collaborative Discussion 2: KQML vs Method Invocation

Initial Post by Marwa Alkuwari: I discussed the advantages of KQML in facilitating semantic communication between autonomous agents, particularly its ability to express intent. While method invocation is more direct and faster, KQML supports reasoning and contextual understanding, essential for open agent environments. Challenges include the need for shared ontologies and parsing complexity.

Peer Response: Rodrigo supported the argument by highlighting how KQML enables dynamic knowledge exchange and supported reasoning mechanisms, despite being harder to implement than traditional function calls.

References

  • Finin, T. et al. (1994). KQML as an Agent Communication Language. In CIKM '94.
  • Labrou, Y., Finin, T., & Peng, Y. (1999). Agent communication languages. IEEE Intelligent Systems, 14(2), 45-52.
  • Mayfield, J., Labrou, Y., & Finin, T. (1996). Evaluation of KQML. In ATAL. Springer, pp. 347–360.

Activity: Deep Learning for Mental Health Diagnosis

Overview: One emerging application of deep learning is its use in diagnosing mental health conditions through voice analysis. AI-powered tools can now detect signs of depression, anxiety, or PTSD based on tone, pitch, speech patterns, and hesitation frequency during spoken conversations.

How It Works: These systems use recurrent neural networks (RNNs) and transformers trained on labeled datasets containing speech samples from both healthy individuals and patients with diagnosed mental conditions. The models learn to identify subtle speech anomalies linked to specific mental states.

Socio-Technical Impact: This technology has strong potential for early detection and intervention, particularly in underserved areas. However, it also raises critical concerns around privacy, consent, data sensitivity, and the potential for algorithmic bias. Ethically, there is a risk of over-reliance on machine judgment without proper clinical validation, and socially, it could impact how employers or insurers perceive individuals flagged by such systems.

This use case bridges intelligent agent research with real-world impact and shows how ethical foresight is essential in deploying deep learning in sensitive domains.
